## Policies

### General

- All code must compile without warnings
- All tests must pass with race detector: `go test -race ./...`
- Files should stay under 200 lines; split as needed
- Commit messages: concise, imperative mood
- Never force-push to `main`
- Docs must update with every behavioral change -- if not updated, work is considered failed
- CHANGELOG.md updated with date + bullet list for every change
- No secrets in code or logs
- Minimize external dependencies (this is a library)

### Go-Specific

- Every exported identifier must have GoDoc comment
- Every file must start with a header comment: `// Package foo: brief description.`
- Prefer `fmt.Appendf` over `fmt.Sprintf` in hot paths
- Use `sync.Pool` for reusable buffers
- No per-log-call heap allocations in steady state
- Single-writer agent goroutine handles all sinks (no races by design)
- Formatting done in agent goroutine, not caller
- `Shutdown(ctx)` must drain queue, flush sinks, close handles, finalize WAV files
- Use interfaces + fakes for hardware dependencies; no test may require real hardware
- Use `t.TempDir()` for file-based tests
- Table-driven tests preferred for multiple input/output cases

## Architecture

```
Caller goroutine(s)           Agent goroutine
  |                              |
  | clog.Info(iface, msg)        |
  |---> [bounded queue] -------->|---> dedupe --> format --> sinks --> hooks
  |                              |
  | clog.Shutdown(ctx)           |
  |---> drain ------------------>|---> flush --> close --> exit
```

### Log Levels

Debug < Info < Success < Warning < Fail < Error < Catastrophe

### Key Features

- Deduplication: consecutive identical lines collapsed, summary on change/shutdown
- Drop policy: `drop_new` (default) or `drop_old` when queue full
- File routing: per-level file output
- Hooks: global and per-level
- Audio: PCM16 frames to WAV files
- Graceful shutdown: drain, flush, close, finalize
